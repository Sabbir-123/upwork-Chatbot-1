One message, one deterministic path — MOCK_LLM=trueOne message, one model call in the middle — MOCK_LLM=false

No model is called in this mode. Intent comes from an ordered regex matcher, every fact comes from a hardcoded lookup, and the whole round trip stays inside one Node process. This traces exactly what runs, in order, for a single POST /api/chat.

Everything is identical to mock mode except one step: intent now comes from a real LLM call to OpenRouter instead of a regex list. Every fact still comes from the same hardcoded lookups — the model only ever picks a label, it never states an order status, policy, or price itself.
